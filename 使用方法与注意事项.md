# ExplorerBlurMica 白条自动修复（方案② 事件驱动）

用脚本自动消除 ExplorerBlurMica 在「浏览器拉起下载目录」时底部出现的白条，
不再需要你每次手动拖动窗口。

---

## 文件清单

| 文件 | 作用 |
|------|------|
| `explorer-blur-fix-event.ps1` | 事件驱动 nudge 脚本本体（SetWinEventHook） |
| `run-nudge-event.bat` | 静默启动器，双击即运行 |

> 旧版轮询脚本 `explorer-blur-fix-nudge.ps1` 仍保留作备选，但**不要和本版同时运行**。

---

## 使用方法

### 1. 启动
双击 `run-nudge-event.bat`。窗口会一闪后隐藏，没有可见界面，正常现象。

### 2. 生效
启动后，**新打开**的资源管理器窗口（含浏览器拉起的下载目录）会被自动 nudge，
白条不再需要你手动拖动。

### 3. 停止
任务管理器 → 找到并结束 `powershell.exe` 进程即可（隐藏运行，无托盘图标）。
若你是从可见的 PowerShell 窗口手动运行脚本，按 `Ctrl+C` 可优雅退出。

### 4. 开机自启（可选）
把 `run-nudge-event.bat` 的快捷方式放进启动文件夹：
「运行」(Win+R) → 输入 `shell:startup` → 打开的文件夹里粘贴快捷方式。
之后每次登录自动运行。

## 注意事项（重要）

1. **必须在你自己机器上双击 `.bat` 运行。**
   绝对不要在 WorkBuddy 的 PowerShell 里执行——那里 `Add-Type` 被沙箱拦截会失败。
   你本机 PowerShell 是 FullLanguage，正常双击运行没问题。`.bat` 已带
   `-ExecutionPolicy Bypass` 兜底执行策略限制。

2. **只跑一个版本。**
   如果之前试过旧的轮询版 `explorer-blur-fix-nudge.ps1`，先结束那个 `powershell`
   进程再跑新版，否则两个会重复 nudge（无害但多余）。

3. **只对「之后新开的」窗口生效。**
   事件驱动只在窗口**创建时**触发，已经打开的旧窗口不会被自动修复。
   启动脚本后，把已有窗口关掉重开（或手动拖一下）即可。

4. **它是「自动复刻你拖一下」，不是根治。**
   ExplorerBlurMica 的 `config.ini` 仍是 `effect=2`，没有改动。
   白条从源头没消失，只是被脚本即时自动修掉，你不用动手了。

5. **每个窗口只 nudge 一次。**
   脚本用去重集合记录已处理的窗口，不会反复抖动。

6. **延迟 450ms 可调。**
   脚本里 `Sleep(450)` 是等窗口先画出来再 nudge。
   若偶尔还瞥见一帧白，可把 `450` 调小（如 `250`）；若觉得抖动太早，调大。
   改完保存、重启脚本生效。

7. **性能开销。**
   常驻一个 `powershell.exe` 进程（约 30–60MB 内存），CPU 平时完全 idle、零轮询——
   比旧轮询版更省。

8. **安全可逆。**
   不碰 ExplorerBlurMica 的 DLL、不改配置、不改系统。结束进程即完全还原，零残留。

## 原理简述

**覆盖范围**：脚本监听所有 `CabinetWClass`（资源管理器）窗口的创建，不限来源——浏览器拉起、手动打开、其它程序 ShellExecute 打开的都包含。判断窗口是否由浏览器拉起不可靠（窗口进程都是 explorer.exe），故采用全覆盖策略；正常窗口被 nudge 一次也无副作用。

ExplorerBlurMica 把 DLL 注入 `explorer.exe`，用 `SetWindowSubclass` 监听 `WM_SIZE`
来重铺 DWM 背景。但浏览器用 `ShellExecute` 拉起的窗口，首绘发生在监听器就绪之前，
底部状态栏那条没被盖住 → 露出白底；你一拖动触发 `WM_SIZE` → 重铺正确。

本脚本用 `SetWinEventHook(EVENT_OBJECT_CREATE, WINEVENT_OUTOFCONTEXT)` 监听新的
`CabinetWClass`（资源管理器）窗口，创建后延迟 450ms 调用 `SetWindowPos` 把窗口**高度 +1px（仅下边框向下探 1px 再弹回，宽度不变）**（等价于手动把底边拖动一次），强制 ExplorerBlurMica 重铺背景。

## 与其它方案对比

| 方案 | 机制 | 性能 | 白条 | 实现门槛 |
|------|------|------|------|----------|
| ① 轮询（旧版） | 每 400ms 扫窗口 | 轻量常驻轮询 | 自动修 | PowerShell，易 |
| **② 事件驱动（本版）** | 窗口创建事件触发 | **零轮询、idle** | 自动修 | PowerShell，易 |
| ③ WH_CBT 全局钩子 | 窗口创建中拦截 | 与②持平 | 几乎零闪现 | 需 C++ 编译 DLL 注入 |
| ④ 改 DLL 本体 | 首绘前铺好背景 | — | 根治 | 需未公开的 MToolBox 源码（锁死） |

在「不碰 C++ / 不碰未公开源码」的约束下，② 是外部脚本的天花板：
最省心、最省 CPU，白条自动化消除。
